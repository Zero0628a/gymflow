\section{Resumen Ejecutivo}

GymFlow es una aplicación móvil orientada a apoyar al usuario durante su sesión de entrenamiento en el gimnasio. Su propósito principal es permitir la consulta rápida de grupos musculares, ejercicios y variantes útiles cuando una máquina o implemento no se encuentra disponible, reduciendo así la improvisación y mejorando la continuidad de la rutina.

El proyecto se desarrolla con React Native y Expo, utilizando servicios de Firebase para autenticación y almacenamiento de información. El presente documento recopila el informe del proyecto \textbf{hasta el cierre del Sprint 1}, por lo que mantiene el análisis general del problema, el diseño del sistema, la arquitectura propuesta y la metodología de trabajo, pero limita la planificación detallada, el backlog comprometido y las historias de usuario al \textbf{sprint 0} y al \textbf{sprint 1}.

\section{Pruebas y Resultados}

\subsection{Plan de Pruebas}

Hasta el cierre del Sprint 1 se considera como prioridad la validación del flujo base de acceso del sistema. En este sentido, las pruebas iniciales se enfocan en verificar la navegación entre registro, inicio de sesión y pantalla principal, así como la coherencia visual de las interfaces y la consistencia de los formularios definidos en los mockups.

\subsection{Resultados Obtenidos}

Como resultado del trabajo acumulado hasta Sprint 1, el proyecto cuenta con una base conceptual y visual clara, una arquitectura definida y un backlog funcional organizado para el incremento inicial. Asimismo, quedan documentadas las historias HU-01, HU-02 y HU-03 junto con sus tareas, criterios de aceptación y recursos visuales de referencia.

\subsection{Mejoras Propuestas}

Para el siguiente incremento se propone continuar con la implementación del núcleo funcional del sistema, incorporando la consulta de músculos, ejercicios y variantes. También se recomienda mantener la trazabilidad entre mockups, código y backlog para que la transición desde la autenticación hacia la funcionalidad principal del producto sea consistente.

\section{Conclusiones y Recomendaciones}

\subsection{Conclusiones}

Hasta el Sprint 1, GymFlow cuenta con un avance documental sólido que cubre el problema del dominio, la propuesta de solución, el diseño de interfaces, la arquitectura tecnológica y la organización del trabajo bajo SCRUM. Este corte deja establecida la base necesaria para continuar con el desarrollo funcional del sistema en los siguientes sprints.

\subsection{Recomendaciones}

Se recomienda que los siguientes incrementos mantengan coherencia con la planificación inicial y con el objetivo principal del proyecto: ayudar al usuario a continuar su entrenamiento cuando no dispone de una máquina específica. También conviene actualizar de forma continua la evidencia documental del avance real del sistema para evitar diferencias entre el informe y la implementación.
